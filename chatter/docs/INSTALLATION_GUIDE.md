# CUICHAT — Installation Guide

## A. Prerequisites

### Backend Requirements
- PHP 8.1+ (Laravel 9 requires PHP 8.0+; 8.1 recommended)
- Composer 2.x
- MySQL 5.7+ or MariaDB 10.3+
- Laravel 9
- Node.js (optional, for asset compilation)
- Git

### Flutter App Requirements
- Flutter SDK 3.5+ (project uses Dart SDK `^3.5.4`)
- Dart SDK (bundled with Flutter)
- Android Studio or VS Code with Flutter extension
- Android SDK (API 21+) for Android builds
- Xcode 14+ for iOS builds (macOS only)
- A physical device or emulator

## B. Laravel Backend Setup

### 1. Clone and Install Dependencies

```bash
cd chatter_backend
composer install
```

### 2. Environment Configuration

```bash
cp .env.example .env
php artisan key:generate
```

### 3. Configure `.env`

Open `.env` and set the following variables:

```dotenv
APP_NAME=CUICHAT
APP_ENV=local
APP_KEY=                        # Generated by php artisan key:generate
APP_DEBUG=true
APP_URL=http://your-server-ip:8000

LOG_CHANNEL=stack
LOG_LEVEL=debug

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=your_database_name
DB_USERNAME=your_database_user
DB_PASSWORD=your_database_password

BROADCAST_DRIVER=log
CACHE_DRIVER=file
FILESYSTEM_DRIVER=local
QUEUE_CONNECTION=sync
SESSION_DRIVER=file
SESSION_LIFETIME=120

MAIL_MAILER=smtp
MAIL_HOST=your_smtp_host
MAIL_PORT=587
MAIL_USERNAME=your_email
MAIL_PASSWORD=your_email_password
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS=noreply@cuichat.online
MAIL_FROM_NAME="CUICHAT"

# SMS Provider (optional — OTP falls back to debug mode if not set)
SMS_PROVIDER=           # Set to "twilio" or "vonage"
TWILIO_SID=             # Required if SMS_PROVIDER=twilio
TWILIO_TOKEN=           # Required if SMS_PROVIDER=twilio
TWILIO_FROM=            # Required if SMS_PROVIDER=twilio (e.g. +1234567890)

VONAGE_KEY=             # Required if SMS_PROVIDER=vonage
VONAGE_SECRET=          # Required if SMS_PROVIDER=vonage
VONAGE_FROM=CUIChatter  # Optional sender name for Vonage
```

### 4. Run Database Migrations

```bash
php artisan migrate
```

This creates all tables including the CUI-specific `registration_otps` table and adds CUI fields to the `users` table.

### 5. Create Storage Symlink

```bash
php artisan storage:link
```

This links `storage/app/public` to `public/storage` so uploaded files are publicly accessible.

### 6. Seed Initial Data (Optional)

```bash
# Import optional seed data for interests and settings
mysql -u your_database_user -p your_database_name < add_interests.sql
mysql -u your_database_user -p your_database_name < cui_optional_seed_data.sql
```

### 7. Create Admin Account

```bash
php add_admin.php
```

Or insert directly into the `admins` table via MySQL.

### 8. Start the Development Server

```bash
php artisan serve --host=0.0.0.0 --port=8000
```

For LAN access from a physical device, use your machine's local IP:

```bash
php artisan serve --host=0.0.0.0 --port=8888
```

## C. Flutter App Setup

### 1. Install Dependencies

```bash
cd chatter_flutter/chatter
flutter clean
flutter pub get
```

### 2. Configure Backend URL

Edit `lib/utilities/const.dart` and set the `baseURL` to point to your backend:

```dart
// For local development (Android emulator):
const String baseURL = "http://10.0.2.2:8000/";

// For local development (physical device on same LAN):
const String baseURL = "http://192.168.x.x:8888/";

// For production:
const String baseURL = "https://cuichat.online/";
```

> **Important:** Always include the trailing slash. The app constructs API URLs as `${baseURL}api/endpoint`.

### 3. Configure Firebase

The app requires Firebase for authentication, Firestore (chat), and FCM (push notifications).

1. Create a Firebase project at https://console.firebase.google.com
2. Add Android and iOS apps to the project
3. Download `google-services.json` and place it in `android/app/`
4. Download `GoogleService-Info.plist` and place it in `ios/Runner/`
5. Enable Authentication (Google, Apple, Email/Password) in Firebase Console
6. Enable Firestore Database
7. Enable Cloud Messaging

### 4. Configure Agora (Audio Rooms)

Edit `lib/utilities/const.dart`:

```dart
const String agoraAppId = 'your_agora_app_id';
const String agoraCustomerId = 'your_agora_customer_id';
const String agoraCustomerSecret = 'your_agora_customer_secret';
```

### 5. Run the App

```bash
# Run on connected device or emulator
flutter run

# Run on specific device
flutter run -d device_id

# Run in release mode
flutter run --release
```

### 6. Build Release APK

```bash
flutter build apk --release
```

The APK will be at `build/app/outputs/flutter-apk/app-release.apk`.

### 7. Build App Bundle (for Play Store)

```bash
flutter build appbundle --release
```

## E. Common Setup Issues

### Issue: `php artisan migrate` fails with "column already exists"
The `RegistrationController` includes a self-healing `ensureUserColumnsExist()` method that silently handles duplicate column errors. If migrations fail, check that you are running them in order.

### Issue: Storage symlink not working on Hostinger
Hostinger shared hosting may not support `php artisan storage:link`. See [Deployment Guide](DEPLOYMENT_GUIDE_HOSTINGER.md) for the manual workaround.

### Issue: OTP not received on phone
If `SMS_PROVIDER` is not set in `.env`, the OTP is only returned in the API response (when `APP_DEBUG=true`) and logged to `storage/logs/laravel.log`. The Flutter app shows a yellow debug banner with the OTP in debug mode.

### Issue: Flutter app cannot connect to local backend
- Ensure the backend is running with `--host=0.0.0.0`
- For Android emulator, use `10.0.2.2` instead of `localhost`
- For physical device, use your machine's LAN IP
- Check that the phone and computer are on the same Wi-Fi network

### Issue: Google Sign-In fails
Ensure the SHA-1 fingerprint of your debug/release keystore is added to the Firebase project settings under Android app configuration.
