## Component Breakdown

### Flutter Mobile App
- Built with Flutter 3.5+ / Dart 3.5.4
- State management: GetX (`get: ^4.7.2`)
- Local storage: GetStorage (`get_storage: ^2.1.1`)
- HTTP client: `http: ^1.5.0` via `ApiService` wrapper
- Navigation: GetX routing with `Get.to()`, `Get.offAll()`
- All API URLs defined in `lib/utilities/web_service.dart`
- All parameter names defined in `lib/utilities/params.dart`
- Backend URL configured in `lib/utilities/const.dart`

### Laravel Backend
- Laravel 9 on PHP 8.1
- RESTful API at `/api/*` (all routes in `routes/api.php`)
- Admin panel web routes at `/` (all routes in `routes/web.php`)
- All API routes protected by `CheckHeader` middleware (validates `apikey: 123` header)
- Admin panel routes protected by `CheckLogin` middleware (session-based auth)
- File uploads stored in `storage/app/public/uploads/`
- Publicly accessible via `public/storage/uploads/` symlink

### MySQL Database
- Hosted on Hostinger shared hosting
- Database name: configured in `.env` as `DB_DATABASE`
- 30+ tables covering users, posts, reels, rooms, music, stories, notifications, and more
- CUI-specific tables: `registration_otps` (added in 2026 migration)

### Firebase
- **Authentication**: Used for Google Sign-In, Apple Sign-In, and email-based social login. Firebase UID is not stored in MySQL; the user's email or identifier is used as the `identity` field.
- **Firestore**: Stores all chat messages in real-time. Chat data is not in MySQL.
- **FCM**: Push notification tokens (`device_token`) are stored in the `users` table and used by the backend to send targeted notifications.

### Agora RTC
- Used for live audio rooms
- Backend generates Agora tokens via `POST /api/generateAgoraToken`
- Token generation uses `app/Class/AgoraDynamicKey/RtcTokenBuilder.php`
- App ID configured in `lib/utilities/const.dart`

## Request Lifecycle

### Standard API Request

```
Flutter App
    │
    ├─ ApiService.call(url, param)
    │       │
    │       ├─ Adds header: apikey: 123
    │       ├─ Adds header: Content-Type: application/json
    │       └─ POST request to cuichat.online/api/{endpoint}
    │
Laravel Backend
    │
    ├─ CheckHeader middleware
    │       └─ Validates apikey header → 401 if missing/wrong
    │
    ├─ Route dispatch → Controller method
    │
    ├─ Controller processes request
    │       ├─ Validates input
    │       ├─ Queries MySQL via Eloquent
    │       └─ Returns JSON response
    │
Flutter App
    │
    └─ ApiService completion callback
            └─ Model.fromJson(response) → UI update
```

### CUI Registration Approval Lifecycle

```
User (Flutter App)
    │
    Step 1: Fill registration form (CuiRegistrationScreen)
    │       └─ POST /api/sendRegisterOtp → OTP sent to phone
    │
    Step 2: Enter OTP (RegisterOtpScreen)
    │       └─ POST /api/verifyRegisterOtp → phone_verified_at set
    │
    Step 3: OTP verified → POST /api/register
    │       └─ User created with approval_status = 'pending'
    │       └─ RegistrationPendingScreen shown
    │
Admin (Web Panel)
    │
    Step 4: Admin reviews at /registrationRequests
    │       └─ Views pending tab with user details
    │
    Step 5a: Admin clicks Approve
    │       └─ POST /approveRegistrationRequest/{id}
    │       └─ approval_status = 'approved', email sent to user
    │
    Step 5b: Admin clicks Reject
    │       └─ POST /rejectRegistrationRequest/{id} (with reason)
    │       └─ approval_status = 'rejected', rejection email sent
    │
User (Flutter App)
    │
    Step 6: User tries to login via POST /api/cuiLogin
            └─ If approved: session created, user enters app
            └─ If pending/rejected: error message shown
```

### Media Upload Lifecycle

```
Flutter App
    │
    ├─ User selects image/video (ImagePicker)
    ├─ Image compressed (quality: 50, max 720px)
    │
    ├─ ApiService.multiPartCallApi(url, param, filesMap)
    │       └─ POST /api/uploadFile (multipart/form-data)
    │
Laravel Backend (PostController.uploadFile)
    │
    ├─ Validates file
    ├─ Stores to storage/app/public/uploads/{filename}
    ├─ Returns: { status: true, data: "uploads/filename.jpg" }
    │
Flutter App
    │
    └─ Stores returned path
    └─ Full URL: baseURL + "storage/" + path
               = https://cuichat.online/storage/uploads/filename.jpg
```

### Session Management

```
Login / Registration
    │
    ├─ API returns User object
    ├─ SessionManager.shared.setUser(user)  → GetStorage.write("user", user.toJson())
    ├─ SessionManager.shared.setLogin(true) → GetStorage.write("isLogin", true)
    │
App Launch (SplashController)
    │
    ├─ SessionManager.shared.isLogin() → reads GetStorage
    ├─ If logged in:
    │       ├─ Fetch fresh profile from API
    │       ├─ Fetch global settings
    │       └─ Navigate based on profile completeness:
    │               isBlock=1 → BlockedByAdminScreen
    │               no interests → InterestScreen
    │               no username → UserNameScreen
    │               no profile photo → ProfilePictureScreen
    │               complete → TabBarScreen
    │
    └─ If not logged in → OnBoardingScreen
```

## Technology Justification

| Technology | Reason for Choice |
|------------|-------------------|
| Flutter | Single codebase for Android and iOS; strong community; GetX for state management |
| Laravel 9 | Mature PHP framework; Eloquent ORM; built-in validation; easy deployment on shared hosting |
| MySQL | Widely supported on shared hosting; relational model suits the data structure |
| Firebase Firestore | Real-time capabilities for chat without building a WebSocket server |
| Firebase FCM | Industry-standard push notification service; free tier sufficient |
| Agora RTC | Reliable low-latency audio SDK; token-based security |
| GetStorage | Lightweight local storage for Flutter; no native dependencies |
| Hostinger | Affordable shared hosting; supports PHP 8.1 and MySQL |
